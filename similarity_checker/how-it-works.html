🚀 Blog Similarity Checker

Internal documentation explaining the technical implementation and architecture of our blog similarity checking system

🔄 How It Works

1

Content Analysis

Your draft content is processed and converted into high-dimensional vector embeddings using OpenAI's advanced language models

Model: text-embedding-3-large
Output: 3072-dimensional vector representation
2

Vector Search

Our system searches through a database of existing blog posts using cosine similarity to find the most semantically similar content

Database: Qdrant Vector Database
Search: Top 10 most similar articles
3

AI Analysis

GPT-4o-mini analyzes the top 3 most similar articles in parallel, providing detailed insights and recommendations

AI Model: GPT-4o-mini
Analysis: Parallel processing for speed
4

Smart Results

Get comprehensive analysis including similarity scores, keyword overlap, and specific rewrite suggestions

Output: Risk assessment + actionable recommendations
Keywords: Top article only (not aggregated)

🛠️ Technical Architecture

🤖

OpenAI GPT-4o-mini

AI model for content similarity analysis. Uses 1500 max tokens, 0.1 temperature for consistency. Processes top 3 articles in parallel

🧠

Text Embeddings

OpenAI text-embedding-3-large model generates 3072-dimensional vectors. Used for semantic similarity search via cosine similarity

🗄️

Qdrant Vector DB

Vector database stores blog post embeddings. Searches return top 10 most similar articles with payload data for analysis

⚡

Parallel Processing

UrlFetchApp.fetchAll() executes multiple GPT API calls simultaneously. Reduces total processing time from sequential to parallel

🎨

Frontend Framework

Vanilla HTML/CSS/JS with CSS Grid, Flexbox, and CSS variables. Responsive design with mobile-first approach

☁️

Google Apps Script

Backend handles API calls, environment variables, and data processing. Deployed as web app with proper CORS handling
