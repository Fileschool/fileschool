Size Calculation Tips

  • 1 MB = 1,048,576 bytes (binary) or 1,000,000 bytes (decimal)
  • 10 MB = 10,485,760 bytes (good for documents)
  • 100 MB = 104,857,600 bytes (good for high-res images)
  • 1 GB = 1,073,741,824 bytes (good for videos)
  • Dimensions: Always use format width,height (e.g., 1920,1080)

About Drag & Drop

The Filestack Drag-and-Drop library makes it incredibly easy to add file upload functionality to your web applications:

Quick Setup (Just 3 Lines!)

  • Add HTML Element: Create a div with class .drop-container
  • Include Library: Load the filestack-drag-and-drop UMD module
  • Initialize: Call new filestackDnD.init(apiKey, element, options)

Key Features

  • File Type Filtering: Accept only specific file types (images, PDFs, etc.)
  • Size Limits: Restrict file sizes to prevent large uploads
  • Multiple Elements: Add drag & drop to multiple elements on one page
  • Event Handling: Listen to progress, success, error events
  • Upload Control: Pause, resume, or cancel uploads programmatically

Common Use Cases

  • Profile Picture Upload: Single file drop zone for avatars
  • Document Management: Accept PDFs, Word docs, spreadsheets
  • Photo Gallery: Multi-file image uploads with previews
  • File Attachments: Email-style file attachment zones
  • Bulk Import: CSV, JSON, or data file uploads

Quick Tips

  • Visual Feedback: Style your drop zone with hover effects and drag-over indicators
  • Accept Patterns: Use MIME types (image/*) for broader matching
  • Size in Bytes: Remember 1MB = 1048576 bytes, 10MB = 10485760 bytes
  • Multiple Zones: Use setElements() to add more drop zones after initialization
  • Event Listeners: Use .on('uploadFileFinish', callback) to track completed uploads
  • Security: Add policy/signature for private uploads or to enforce file restrictions server-side

POST /api/store/{provider}

Store / Upload File

Uploads a file directly to any supported storage backend. Returns JSON metadata including a Filestack URL.

Your Generated Command

Upload from URL:

curl -X POST \
  -d url="https://example.com/image.png" \
  "https://www.filestackapi.com/api/store/S3?key=YOUR_API_KEY"

Upload Binary File:

curl -X POST \
  --data-binary @file.png \
  --header "Content-Type:image/png" \
  "https://www.filestackapi.com/api/store/S3?key=YOUR_API_KEY"
Available Parameters
Parameter Type Description
key * string Filestack API key (required)
filename string Name for stored file
path string Path within bucket
container string Bucket/container name
access string S3: "public" or "private"
Example Response (200 OK) 
{
  "url": "https://cdn.filestackcontent.com/s7tdGfE5RRKFUxwsZoYv",
  "size": 8331,
  "type": "image/png",
  "filename": "watermark.png",
  "key": "a1RyBxiglW92bS2SRmqM_watermark.png"
}
GET /api/file/{handle}/metadata

Get File Metadata

Retrieves JSON metadata for an uploaded file. Use your file handle from the configuration above.

Your Generated Command

curl -X GET "https://www.filestackapi.com/api/file/YOUR_HANDLE/metadata"
Available Parameters

Common: size, mimetype, filename, uploaded, writeable, location, path, container

Hashing: md5, sha1, sha224, sha256, sha384, sha512

Example: ?size=true&mimetype=true&filename=true

Example Response (200 OK) 
{
  "mimetype": "image/png",
  "uploaded": 1431950945811.783,
  "container": "fp-documentation-assets",
  "writeable": true,
  "filename": "dtKZNd1J.png",
  "location": "S3",
  "key": "kWg7nloGTWmHFi5nlbF9_dtKZNd1J.png",
  "path": "kWg7nloGTWmHFi5nlbF9_dtKZNd1J.png",
  "size": 270
}
POST /api/file/{handle}

Overwrite File

Overwrites existing file with new content. Requires security (Basic Auth with app secret from top bar).

Your Generated Command

curl -X POST \
  --data-binary @file.txt \
  --header "Content-Type:text/plain" \
  -u "app:YOUR_APP_SECRET" \
  "https://www.filestackapi.com/api/file/YOUR_HANDLE"
Example Response (200 OK) 
{
  "url": "https://cdn.filestackcontent.com/Bc2FQwXReueTsaeXB6rO",
  "mimetype": "plain/text",
  "isWriteable": true,
  "size": 19,
  "filename": "filename.txt"
}
DELETE /api/file/{handle}

Delete File

Deletes a file permanently. Requires security (Basic Auth with app secret).

Your Generated Command

curl -X DELETE \
  -u "app:YOUR_APP_SECRET" \
  "https://www.filestackapi.com/api/file/YOUR_HANDLE"

Tip: Add ?skip_storage=true to remove from Filestack but keep in storage.

GET /api/file/{handle}

Download File

Recommended: Use CDN URLs for better performance and lower costs.

What is Safe for Work?

Safe for Work (SFW) detection uses AI to analyze images and return a simple boolean indicating if content is appropriate for workplace environments.

  • Direct API - Synchronous operation (instant results)
  • Workflows - Also available for automated pipelines
  • Chainable - Combine with other transformations
  • Security Required - Needs policy & signature

Response Format

Safe Image: {"sfw": true}
Unsafe Image: {"sfw": false}

Use Cases

Compliance - Ensure workplace-appropriate content
Conditional Logic - Route files based on SFW status

What Are Workflows?

Workflows automate complex file processing with business rules and conditional logic. Create workflows in the Developer Portal to:

  • Transformations - Image/video/audio processing
  • Intelligence - Tagging, SFW detection, OCR, virus scanning
  • Conditional Logic - Route files based on content or metadata
  • Webhooks - Receive results asynchronously
  • Multi-step - Chain multiple operations together

How It Works

1. Create Workflow - Build in Developer Portal
2. Get UUID - Copy your Workflow ID
3. Run Workflow - Make GET request to start processing
4. Get Job ID - Response includes Job ID
5. Check Status - Poll for results or use webhooks

Important Notes

Workflows are asynchronous - results come via webhook or status check
Security: Requires runWorkflow and convert policy calls
Each upload triggers one workflow job
