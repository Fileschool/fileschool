What is Safe for Work?

Safe for Work (SFW) detection uses AI to analyze images and return a simple boolean indicating if content is appropriate for workplace environments.

  • Direct API - Synchronous operation (instant results)
  • Workflows - Also available for automated pipelines
  • Chainable - Combine with other transformations
  • Security Required - Needs policy & signature

Response Format

Safe Image: {"sfw": true}
Unsafe Image: {"sfw": false}

Use Cases

User-Generated Content - Filter uploads automatically
Moderation - Flag content for manual review
Compliance - Ensure workplace-appropriate content
Conditional Logic - Route files based on SFW status

Security Requirement

SFW detection requires security policy and signature. Make sure to configure these in the API Key section at the top of the page.

OCR Configuration

Required for OCR. Format: p:<base64_policy>,s:<signature>

About OCR

OCR is implemented as URL-based transformations that can be chained with other processing tasks:

  • Extract text from images and PDFs
  • Multi-language support
  • Document detection preprocessing
  • Works with external URLs
  • Chainable with other transformations

URL Transform Examples

Basic OCR:
/security=p:POLICY,s:SIGNATURE/ocr/HANDLE

With Document Detection:
/doc_detection=coords:false,preprocess:true/ocr/HANDLE

External URL:
/APIKEY/security=p:POLICY,s:SIGNATURE/ocr/EXTERNAL_URL

What Are Workflows?

Workflows automate complex file processing with business rules and conditional logic. Create workflows in the Developer Portal to:

  • Transformations - Image/video/audio processing
  • Intelligence - Tagging, SFW detection, OCR, virus scanning
  • Conditional Logic - Route files based on content or metadata
  • Webhooks - Receive results asynchronously
  • Multi-step - Chain multiple operations together

How It Works

1. Create Workflow - Build in Developer Portal
2. Get UUID - Copy your Workflow ID
3. Run Workflow - Make GET request to start processing
4. Get Job ID - Response includes Job ID
5. Check Status - Poll for results or use webhooks

Important Notes

Workflows are asynchronous - results come via webhook or status check
Security: Requires runWorkflow and convert policy calls
Each upload triggers one workflow job
